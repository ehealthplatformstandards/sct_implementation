# SNOMED CT Implementation Repository

Welcome to the **SNOMED CT Implementation Repository**.

This repository is intended to collect questions, implementation issues, improvement proposals and feedback related to the implementation of **SNOMED CT** in Belgium.

Whether you are a software vendor, healthcare professional, terminology expert, analyst or project manager, your feedback is valuable and helps improve the overall quality and consistency of the implementation.

---

# Repository objectives

This repository is used to:

- collect implementation questions;
- report issues or inconsistencies;
- propose improvements;
- discuss implementation decisions;
- document implementation experience;
- facilitate collaboration between all stakeholders.

This repository **is not intended to report software bugs** in individual products.

---

# Who can contribute?

Anyone involved in the implementation of SNOMED CT in Belgium, including:

- Software vendors
- Healthcare organizations
- Government organizations
- Analysts
- Developers
- Clinical experts
- Terminology experts
- Project managers

---

# Before creating an Issue

Before submitting a new Issue, please:

- search whether the same question has already been asked;
- verify whether the topic has already been answered;
- keep one Issue per topic.

This greatly simplifies discussions and follow-up.

---

# How to create an Issue

If you have never used GitHub before, don't worry.

Creating an Issue only takes a few minutes.

1. Open the Issues page:

https://github.com/ehealthplatformstandards/sct_implementation/issues

2. Click on the green **New issue** button.

3. Choose the appropriate Issue template (if available).

4. Complete the requested information.

5. Click **Submit new issue**.

Your Issue will automatically become visible to the implementation team.

---

# Issue title

Please use a clear and concise title.

Good examples:

```
Concept missing for vaccination procedure

Question regarding inactive concepts

Mapping issue between ICD-10 and SNOMED CT

Clarification on post-coordination

Reference Set update request
```

Avoid titles such as:

```
Problem

Question

Help

SNOMED

Issue
```

A descriptive title makes searching much easier.
